# Agentic tools

Registry and pipeline commands for **AI agents** (and humans) working in this repo. Use these instead of improvising edits to generated files.

**Start here:** [registry.json](registry.json) (machine-readable) · [workflows/codegen.md](workflows/codegen.md) (step-by-step)

## Non-negotiable rules

1. **Never hand-edit** `*.g.cs` under `src/Novolis.Raylib.Bindings/Interop/` or `src/Novolis.Raylib.Runtime/` (files contain `<auto-generated />`).
2. **Manifests are source of truth** — `codegen/pipeline/raylib6/*.manifest.json`.
3. **After manifest or hook changes**, run `codegen.generate` then `agent.verify` before finishing the task.
4. **`src/`** = publishable NuGet packages only. **`codegen/`** = build tooling (never published).

## Quick commands

| Tool id | Command |
|---------|---------|
| `pipeline.run` | `dotnet run --project codegen/Novolis.Raylib.Pipeline -- run maintainer` |
| `codegen.generate` | `dotnet run --project codegen/Novolis.Raylib.Pipeline -- run generate` |
| `codegen.drift-check` | `dotnet run --project codegen/Novolis.Raylib.Pipeline -- run ci-codegen` |
| `agent.verify` | `dotnet run --project codegen/Novolis.Raylib.Pipeline -- run agent-verify` |

## Cursor integration

- Rules: `.cursor/rules/` (codegen + generated-file guards)

- Root guide: [AGENTS.md](../AGENTS.md)

## Adding a tool

Add an entry to `registry.json` and a one-line note in this README. Prefer `dotnet run --project codegen/Novolis.Raylib.Pipeline` profiles over one-off shell wrappers.
