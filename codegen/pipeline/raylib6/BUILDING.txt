raylib 6 + raygui pipeline (Novolis.Raylib)
=========================================

Linear step pipeline with committed `result.json` + `step.log` per step under
`codegen/pipeline/raylib6/steps/`. Heavy artifacts (DLLs, clones) go in each step's
`artifacts/` folder (gitignored).

List steps:

  dotnet run --project codegen/Novolis.Raylib.Pipeline -- list

Full maintainer flow (fetch + native + codegen + drift):

  dotnet run --project codegen/Novolis.Raylib.Pipeline -- run maintainer

Agent gate (CI parity + build):

  dotnet run --project codegen/Novolis.Raylib.Pipeline -- run agent-verify

Extend codegen behavior with hooks in codegen/Novolis.Raylib.CodeGen.Hooks/
(see codegen/pipeline/raylib6/hooks/README.md).

Steps
-----

1) step_01_source — vendor headers + prebuilt raylib, raygui.h, raylib-cimgui
   dotnet run --project codegen/Novolis.Raylib.Pipeline -- run step_01_source

2) step_02_native — CMake shims (trace, ImGui, raygui) → step_02_native/artifacts/
   dotnet run --project codegen/Novolis.Raylib.Pipeline -- run step_02_native

3) step_03_verify_manifest — manifest imports vs raylib.h

4) step_04_enrich_docs / step_05_verify_docs — façade XML doc enrichment

5) step_06_codegen — Raylib6Native.g.cs, façades, Hud, Gui, RayGui (*.g.cs)

6) step_07_drift — git diff on pipeline manifests + generated C#

7) step_08_build — Release build Bindings + Runtime (agent-verify profile)

Outputs
-------
- codegen/pipeline/raylib6/steps/step_01_source/artifacts/raylib-6/
- codegen/pipeline/raylib6/steps/step_02_native/artifacts/*.dll
- codegen/pipeline/raylib6/*.manifest.json → src/**/Interop/*.g.cs and Runtime/**/*.g.cs

Build .NET:

  dotnet build Novolis.Raylib.slnx
